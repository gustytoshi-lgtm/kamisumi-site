# KAMISUMI ローカル動作確認ガイド（はじめての方向け）

このガイドは、プログラミングに詳しくない方が、自分のパソコン（Windows）で
KAMISUMI のサイトと管理画面を**起動・確認・初期化**するための手順です。

> ⚠️ ここで触るのは「mock（サンプル）モード」です。**本物の顧客データや本番サイトではありません。**
> 入力した内容は**サーバーを止めると消えます**（練習用）。安心して試してください。

---

## 0. 用語（かんたん）

- **公開サイト**: お客様が見るページ（日本語 / 繁體中文）。
- **管理画面**: 商品・在庫・注文などを管理する裏側のページ。
- **mock モード**: 練習用のサンプルデータで動くモード。
- **ロール**: 管理画面での権限。owner（全部）/ front_staff（接客）/ inventory_staff（在庫）。

---

## 1. 初回だけ必要な準備

1. **Node.js** をインストールします（1回だけ）。
   - https://nodejs.org/ を開き、「LTS」版をダウンロードしてインストール。
2. 確認は不要です。起動ファイルが自動で必要な部品（npm install）を入れます（初回は数分）。

> うまくいかないときは、起動ファイルが日本語でエラーを表示し、ウィンドウは開いたままになります。

---

## 2. 起動（ダブルクリックするだけ）

プロジェクトフォルダ直下のファイルをダブルクリックします。

| ファイル | 何が開くか |
|---|---|
| `START_KAMISUMI_MOCK.cmd` | 管理画面 + **動作確認ページ**（おすすめ・最初はこれ） |
| `START_KAMISUMI_OWNER.cmd` | owner（全権限）で管理画面 |
| `START_KAMISUMI_FRONT_STAFF.cmd` | front_staff（接客向け。原価・利益は非表示） |
| `START_KAMISUMI_INVENTORY.cmd` | inventory_staff（在庫向け） |
| `START_KAMISUMI_PUBLIC_ONLY.cmd` | 公開サイトのみ（お客様の見え方） |

- 黒いウィンドウが開き、「準備完了」と出ると**自動でブラウザが開きます**（少し待ちます）。
- すでに起動中のときは、二重に起動せずブラウザだけ開きます。
- **このウィンドウを閉じるとサーバーが止まります。**

### ロール別の見え方を確認したいとき
役割ごとの起動ファイル（OWNER / FRONT_STAFF / INVENTORY）を使い分けてください。
front_staff では「原価・利益・権限管理・全顧客CSV」は表示されません。

---

## 3. 止める

- かんたん: **起動した黒いウィンドウを閉じる**。
- または `STOP_KAMISUMI.cmd` をダブルクリック（このランチャーで起動したサーバーだけを安全に止めます）。

---

## 4. サンプルデータを初期状態に戻す

1. サーバーを起動した状態で `RESET_MOCK_DATA.cmd` をダブルクリック。
2. 確認（y）を入力するとリセットされます。
3. もしくは、動作確認ページの「mock データをリセット」ボタン。
4. ※ サーバーを再起動するだけでも初期状態に戻ります。

---

## 5. よく使う URL

起動後、ブラウザのアドレス欄に入れて確認できます（ポートは通常 3000）。

- 公開（日本語）: `http://localhost:3000/ja`
- 公開（繁體中文）: `http://localhost:3000/zh-tw`
- 管理トップ: `http://localhost:3000/ja/admin`
- **動作確認ページ**: `http://localhost:3000/ja/admin/dev-check`
- 商品: `/ja/admin/products` / 在庫: `/ja/admin/inventory` / 注文: `/ja/admin/orders`
- 買付: `/ja/admin/sourcing` / Journal: `/ja/admin/journal`
- 業務設定: `/ja/admin/settings` / 仕入先: `/ja/admin/suppliers`
- 入金: `/ja/admin/payments` / 配送: `/ja/admin/shipping`
- 繁體中文の管理画面: 同じ URL の `/ja/` を `/zh-tw/` に変えるだけでは UI 言語は変わりません（管理画面の言語は利用者設定です）。
  繁體中文 UI を確認したい場合は、起動前に環境変数 `ADMIN_DEV_LOCALE=zh-tw` を設定してください
  （例: コマンドプロンプトで `set ADMIN_DEV_LOCALE=zh-tw` のあと `npm run dev:owner`）。既定は日本語です。

---

## 6. 動作確認ページ（dev-check）

`START_KAMISUMI_MOCK.cmd` で開くページです。次が一目で分かります。

- 今の環境（mock / 本番、認証、ロール、管理画面の有効/無効）
- サンプルデータの件数、migration（DB設計）の一覧
- 実装済み / 未実装の画面
- 公開・管理ページを新しいタブで開く**ワンクリックリンク**
- サンプルデータのリセットボタン

緑（✅正常）/ こげ茶（要確認）/ 灰（未設定）/ 金（未実装）で色分けされます。

---

## 7. かんたんな自動チェック

`CHECK_KAMISUMI.cmd` をダブルクリックすると、1〜3分で主要ページを自動確認し、
`✅ 正常` / `❌ エラー` を日本語で表示します（公開・管理・日本語・繁體中文・権限制限）。
※ これは別ポート(3100)で一時的にサーバーを起動して確認し、自動で停止します。

## 8. 商品・在庫・注文などの操作（管理画面）

- **商品**: 一覧から状態（在庫あり/完売など）を変更できます。
- **在庫**: 在庫アイテムを作り、入荷・予約・出荷などの「在庫移動」を登録します（数量は履歴付きで増減）。
- **注文**: 仮注文を作成し、状態（問い合わせ→見積→入金→発送…）を進めます。順番のルールは自動でチェックされます。
- **買付 / Journal**: 依頼の受付状態、記事の下書き・公開などを操作できます。
- **業務設定**（owner）: 問い合わせメール・SNS URL・取り置き時間・注文/再入荷の受付などを編集できます。変更履歴も表示されます。API鍵・口座番号などシステムに関わる項目は安全のため編集できません。
- **仕入先**（owner）: 仕入先の登録・公開レベル変更・削除/復元。連絡先や内部メモは公開サイトには出ません。
- **入金**（owner）: 入金記録の作成・状態変更・入金額の記録。金額は「980.50」のように普通の単位で入力できます。実際の銀行口座番号は保存しません。
- **配送**（owner / front_staff）: 配送の作成・状態変更（梱包→発送→配達など）。実送料と顧客請求送料を記録できます。
- 危険な操作（削除・キャンセル・在庫減算・状態確定・リセット等）には確認ダイアログが出ます。
- 権限のない画面（例: front_staff で入金・仕入先・設定）を開くと「権限がありません」と表示されます。

> ⚠️ mock モードでは、管理画面での変更は**公開サイトには反映されません**（練習用の別データのため）。
> 本番（Supabase）接続時には、両者が同じデータになります。

- 管理画面: ログインユーザーの設定言語（ja / zh-tw）で表示されます。

## 10. よくあるエラーと対処

| 症状 | 対処 |
|---|---|
| 「Node.js が見つかりません」 | https://nodejs.org/ の LTS をインストール |
| ブラウザが開かない | 黒いウィンドウに出る `http://localhost:3000/...` を手動で開く |
| 「ポートは使用中」 | すでに起動済み。`STOP_KAMISUMI.cmd` で止めてから再起動 |
| 画面が真っ白／崩れる | 起動直後は準備中のことがある。少し待って再読み込み |
| 変更が公開サイトに出ない | mock の仕様です（第8章の注意）。本番接続で解消 |

## 11. OneDrive と「緑チェック」について

- このフォルダは OneDrive 内にあります。**同期中はファイルがロックされ、起動やビルドが失敗することがあります。**
- 失敗したら少し待つか、`.next` フォルダを削除して再実行してください（`.next` は自動で作り直されます）。
- 開発者向け: コミット前は `npm run verify:full`（型・lint・テスト・DB検証・ビルド）が**すべて緑**になるのを待ってください。途中の状態で配布しないためです。

---

## 12. 本番との違い・やってはいけないこと

- ここは**練習用 mock** です。実際の在庫・注文・入金ではありません。
- **やってはいけない**: `.env.local` や秘密鍵をメール/チャットで共有する、OneDrive の同期設定を勝手に変える、`node_modules` や `.next` を手で編集する。
- 本番（Supabase）接続は開発者が `docs/SUPABASE_SETUP.md` の手順で行います。本番では開発用機能（動作確認ページ・ロール切替・リセット）は**自動的に無効**になります。
